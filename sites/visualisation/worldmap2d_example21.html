Example 21: Show model data added by url

With the addModelFile function you simply add geo data files to the map.
Try it with: ../../data/worldmap/example3/stadtgebiet.geojson (Displays the city borders of Bielefeld)
And with: ../../data/worldmap2d/shapefiles/congress.zip (Asks for the type. Choose shapefile here)
And with: ../../data/worldmap2d/geotiffs/example_australia.tif (Displays Geotiff in Australia)
And with: https://geoportal.bafg.de/arcgis1/rest/services/INSPIRE/Catchment/MapServer/exts/InspireView/service?VERSION=1.3.0&SERVICE=WMS&REQUEST=Getcapabilities (Displays water bodies)
And with: https://api.openweathermap.org/data/2.5/box/city?bbox={lon-left},{lat-bottom},{lon-right},{lat-top},{zoom}&appid=API-KEY (viewport based GeoJSON, you must provide a valid API key)

WMS Support notes

WMS (WebMapService) is a widly used standard for requesting tiles from geo information serves (such as arcgis). SWAC currently supports following WMS urls:

Capabilities-Link XML format

If a link to a capabilities XML file (like example )is provied, a popup displayes the known layers and let the user choose which of them to view.

Capabilities-Link JSON format

If a link to a capabilities JSON file (like example )is provied, a popup displayes the known layers and let the user choose which of them to view.

Tile-Link example

If a link to a request of a concrete tile (like example) is given, the WMS Format plugin tries to build the neccessery modification for requesting leaflet supporting tiles.

Limitations

WMS support is currently limited to tiles and Legends. Data fetch are currently not supported.
